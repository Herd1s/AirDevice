# AirDevice 超详细新手说明书

适用对象：
- 第一次做单片机项目
- 第一次接蓝牙串口模块（JDY23）
- 第一次做“传感器 + OLED + 按键 + 手机APP”整合项目

本说明书目标：
- 让你从 0 到 1 接好线、跑起来、看见数据
- 明白每个模块到底在做什么
- 明白为什么会报警、为什么风扇会开关
- 明白手机和单片机怎么通信、每条命令有什么用

---

## 1. 项目一句话解释

AirDevice 是一个空气监测控制系统。
STM32 负责采样、计算、显示和控制，JDY23 负责蓝牙透传，手机APP负责查看数据、改阈值和看日志。

## 2. 系统整体结构（先建立大脑地图）

```
   +------------+------------+--------------+
                        |
                    [STM32]
                 /      |      \
            [OLED]   [按键]   [继电器/蜂鸣器]
                               |
                             [风扇]
                        |
                    [USART2]
                        |
                     [JDY23]
                        |
                    (Bluetooth)
                        |
                     [手机APP]
```

## 3. 你会得到哪些功能

- 实时显示：温湿度、eCO2、TVOC、PM2.5、MQ135、AQ分数
- 本地按键设置阈值
- 超阈值报警（蜂鸣器）
- 超阈值联动风扇（继电器）
- 手机蓝牙查看实时数据
- 手机蓝牙改阈值与控制模式
- 报警日志记录与读取
- 参数断电保存（Flash）

## 5.2 电源接线规则（非常重要）

- 所有模块必须共地：`GND` 全连在一起
- SGP30、OLED、JDY23 推荐 3.3V 供电
- MQ135、GP2Y1014 常见模块会用到 5V（请按你手上模块丝印）
- 风扇和继电器电源按负载规格，不要硬上板载 3.3V

## 5.3 电平安全规则（烧板高发区）

- `PA4/PA5` 是 STM32 ADC，输入电压不能超过 3.3V
- 如果 MQ135/GP2Y 的 AO 超过 3.3V，需要分压后再接
- JDY23 串口电平要匹配（常见 3.3V TTL）

## 6. 先接谁后接谁（新手推荐顺序）

1. 先接 STM32 + OLED + 按键 + 蜂鸣器（不接风扇）
2. 上电确认 OLED 正常显示
3. 再接 DHT11、SGP30
4. 再接 MQ135、GP2Y1014
5. 最后接 JDY23 蓝牙
6. 最后再接继电器负载（风扇）

这样出问题时，你能快速定位是哪一段新增接线导致的。

## 7. 固件运行节拍（你看到的数据多久更新一次）

程序主循环 10ms 执行一次。

| 任务 | 周期 | 内容 |
|---|---|---|
| 按键扫描 | 10ms | 消抖、短按/长按识别 |
| 模拟采样任务 | 500ms | MQ135 + PM2.5 + AQ判定 |
| DHT11 任务 | 2s | 温湿度更新 |
| SGP30 任务 | 1s | eCO2/TVOC 更新 |
| OLED 刷新 | 200ms | 页面刷新 |
| BLE主动上报 | 1s | 发送 `$DATA,...` |

## 8. 每个模块的工作原理（通俗解释）

## 8.1 DHT11

- 类型：单总线数字传感器
- 作用：给出温度和湿度
- 代码行为：每 2 秒读一次
- 典型故障：DATA 线没拉好、供电不稳会导致读不到

## 8.2 SGP30

- 类型：I2C 数字空气质量传感器
- 地址：`0x58`
- 输出：`eCO2(ppm)`、`TVOC(ppb)`
- 代码行为：
  - 先发 `0x2003` 进行 IAQ 初始化
  - 周期发 `0x2008` 获取空气质量
  - 对每 2 字节数据做 CRC-8 校验（多项式 0x31，初始 0xFF）
- 如果 CRC 错误，数据会丢弃

## 8.3 MQ135

- 类型：模拟传感器
- 输出：原始 ADC 值（不是标准 ppm）
- 代码行为：500ms 采样，做滤波和限幅
- 注意：项目里用它做“相对空气质量趋势指标”

## 8.4 GP2Y1014AU0F

- 类型：粉尘传感器
- 特点：需要控制内部 LED 点亮后在窗口期采样
- 代码行为：
  - 先拉 LED 控制（低电平有效）
  - 延时后采样 ADC
  - 再关 LED
  - 用近似公式转 PM 值

## 8.5 OLED

- 类型：SSD1306 I2C 显示器
- 作用：本地显示数据和设置界面
- 接线：PB6/PB7，和 SGP30 分总线

## 8.6 蜂鸣器和继电器

- 都是低电平触发
- 蜂鸣器在报警且未静音时按节拍鸣叫
- 继电器由风扇状态决定（自动或手动）

## 8.7 JDY23

- 本质：蓝牙到 UART 的“无线串口线”
- 不负责算法和判断，只负责转发文本命令和回包

## 9. 数据处理逻辑（为什么看起来比较稳定）

程序为了防抖做了 3 层处理：

1. 中值均值（7 次采样去掉最小和最大，取中间 5 个平均）
2. 滑动平均（长度 8）
3. 限幅（每次变化最大步进限制）

作用：
- 抗瞬时毛刺
- 防止数值跳变太快
- 页面显示更稳定

---

## 10. AQ 分数与报警判定（核心逻辑）

## 10.1 AQ 分数

当 SGP30 在线时：

`score = 35%*PM比例 + 35%*CO2比例 + 20%*TVOC比例 + 10%*MQ比例`

当 SGP30 离线时：

`score = 70%*PM比例 + 30%*MQ比例`

比例定义：
- 比例 = 当前值 / 对应阈值 * 100

## 10.2 AQ 等级

| 分数区间 | 等级 |
|---|---|
| < 80 | Excellent |
| 80 ~ 119 | Good |
| 120 ~ 169 | LightPoll |
| 170 ~ 229 | Moderate |
| >= 230 | HeavyPoll |

## 10.3 报警触发与解除

- 任一关键项超过阈值 -> 报警触发
- 报警解除不是“立刻”解除，使用回差（hysteresis）防抖：
  - PM2.5 回差：10
  - CO2 回差：200
  - TVOC 回差：50
  - MQ135 回差：100

## 10.4 预热策略

- MQ135 预热计时：45 秒（预热期内不参与超限触发）
- SGP30 预热提示：15 秒（显示在页面）

---

## 11. 风扇与蜂鸣器控制逻辑

## 11.1 风扇

- 自动模式（AUTO）：
  - 报警时风扇立即开
  - 报警解除后不立刻关，延时 `FANOFF` 秒后关
- 手动模式（MAN）：
  - 风扇由人工开关，不受自动判定影响

## 11.2 蜂鸣器

- 条件：`报警=ON` 且 `静音=OFF`
- 声音节拍：约 250ms 翻转一次（间歇鸣叫）

---

## 12. OLED 页面详细解释

## 12.1 页面 0：状态总览

- `T1/H1`：温湿度（DHT11）
- `CO2`：eCO2 当前值
- `FAN`：AUTO/MAN + ON/OFF
- `ALM`：报警状态；`M`：静音状态

## 12.2 页面 1：关键污染指标

- `CO2`
- `TVOC`
- `PM25`
- `SGP30 Online/Offline`

## 12.3 页面 2：综合评估

- `MQ135` 原始值
- `AQ` 分数 + 文字等级
- `SGP30` 在线状态
- `SGPWarm` 或 `Warmup` 或 `FanOffDelay`

## 12.4 设置页

- 第一行提示正在设置阈值
- 第二行显示当前参数和值
- 第三行提示 K2/K3 调整、K1 切换参数
- 第四行提示 K4 保存

## 13. 按键功能（每个动作都写清楚）

长按判定：约 800ms。

| 按键 | 短按 | 长按 |
|---|---|---|
| K1 | 运行模式切页；设置模式切参数项 | 进入/退出设置模式 |
| K2 | 设置模式 +1 | 设置模式 +10 |
| K3 | 设置模式 -1 | 设置模式 -10 |
| K4 | 设置模式下保存；运行模式下静音切换或模式切换 | 手动模式下强制开关风扇 |

补充说明：
- 若当前报警中，K4 短按优先执行“静音切换”
- 若不报警，K4 短按执行 AUTO/MAN 切换

## 15. 蓝牙通信原理（彻底讲清）

## 15.1 传输通道

- 物理：手机蓝牙 <-> JDY23
- 逻辑：JDY23 串口 <-> STM32 USART2
- 参数：9600 波特率，ASCII 文本

## 15.2 协议格式

- 一条命令一行
- 行结束符：`\n`
- 字段分隔：`,`
- 设备主动回包以 `$` 开头

## 15.3 设备会主动发什么

- 上电后：
  - `$BOOT,AirDevice,JDY23,READY`
  - `$CFG,...`
- 每秒：
  - `$DATA,...`
- 报警状态变化时：
  - `$EVT,ALARM,ON,...` 或 `$EVT,ALARM,OFF,...`

## 15.4 支持命令总表

| 类别 | 命令 | 说明 |
|---|---|---|
| 心跳 | `PING` | 连通测试 |
| 查询 | `GET,DATA` | 获取当前数据 |
| 查询 | `GET,CFG` | 获取阈值配置 |
| 查询 | `GET,LOG` | 获取设备报警日志 |
| 查询 | `GET,ALL` | 获取数据+配置 |
| 设置 | `SET,CO2,<v>` | 设置 CO2 阈值 |
| 设置 | `SET,TVOC,<v>` | 设置 TVOC 阈值 |
| 设置 | `SET,PM25,<v>` | 设置 PM2.5 阈值 |
| 设置 | `SET,MQ135,<v>` | 设置 MQ135 阈值 |
| 设置 | `SET,FANOFF,<v>` | 设置风扇延时关断 |
| 设置 | `SET,MODE,AUTO` | 自动模式 |
| 设置 | `SET,MODE,MAN` | 手动模式 |
| 设置 | `SET,FAN,1/0` | 手动模式下强制风扇开关 |
| 设置 | `SET,MUTE,1/0` | 报警静音开关 |
| 维护 | `SAVE` | 保存参数到 Flash |
| 维护 | `CLR,LOG` | 清空设备日志 |

## 15.5 ACK 错误码解释

| 返回 | 含义 |
|---|---|
| `ERR,ARG` | 参数缺失 |
| `ERR,CMD` | 不支持的命令 |
| `ERR,VAL` | 数字格式错误 |
| `ERR,RANGE` | 超出允许范围 |
| `ERR,MODE` | 当前模式不允许该操作 |
| `ERR,FLASH` | Flash 保存失败 |

## 16. 报警日志机制

- 设备端环形日志长度：20 条
- 每次报警状态变化（OFF->ON 或 ON->OFF）写一条
- 每条日志包含：
  - 运行时间
  - 报警状态
  - AQ 分数
  - CO2/TVOC/PM25/MQ 快照
- `GET,LOG` 会先回 `COUNT`，再逐条发，最后 `$ENDLOG`

## 18. 从零上手的最稳流程（强烈推荐）

1. 只接 STM32 + OLED + 按键，上电看屏是否正常
2. 加 DHT11 + SGP30，看数据页是否变化，SGP30 是否 Online
3. 加 MQ135 + GP2Y，看 PM/MQ 是否有值
4. 加蜂鸣器，不接风扇，先验证报警逻辑
5. 加 JDY23，手机连蓝牙，发 `GET,ALL` 验证通信
6. 最后加继电器和风扇负载

## 19. 自检清单（每次调试都能用）

- 是否全部共地
- 是否把 TX/RX 交叉连接（PA2->JDY RX，PA3<-JDY TX）
- ADC 输入是否可能超过 3.3V
- SGP30 是否接在 PB10/PB11（不是 PB6/PB7）
- 按键是否都是“GPIO 到 GND”的接法
- 蜂鸣器/继电器是否为低电平触发类型
- 是否发送了换行结尾命令（`\n`）

## 20. 常见问题与处理（更细版）

## 20.1 蓝牙连接了但没数据

- 问题点：
  - PA2/PA3 反接
  - 波特率不一致
  - 命令没换行
  - JDY23 未真正连上（只配对未建立数据通道）
- 处理：
  - 先发 `PING`
  - 看是否收到 `$ACK,PING,OK`

## 20.2 SGP30 离线

- 问题点：
  - PB10/PB11 接错
  - 电源不稳
  - 总线无上拉（部分裸模块）
- 处理：
  - 查电源和地
  - 查 SCL/SDA 连续性
  - 重新上电观察 `SGP30:Online`

## 20.3 报警一直响

- 可能原因：
  - 阈值设太低
  - 环境值真实超限
  - 传感器未预热稳定
- 处理：
  - 先按 K4 静音
  - 查看数据页面
  - 合理提高阈值再观察

## 20.4 风扇不动

- 先看屏幕 `FAN:... ON/OFF`
- 如果屏幕 ON 但风扇不转：
  - 继电器供电与触发电平不匹配
  - 继电器输出接线错误
  - 风扇电源路径错误

  - 默认参数加载
  - Flash 参数读取与校验
  - Flash 参数写入保存
- 负责阈值和风扇延时的持久化。

`STM32/User/app_sensor.c`
- 传感器与算法核心：
  - DHT11 读取
  - SGP30 I2C 通信与 CRC 校验
  - MQ135 / GP2Y1014 ADC 采样
  - 滤波（中值均值 + 滑动平均 + 限幅）
  - AQ 分数计算与报警判定
- 如果后续要加新传感器，优先改这个文件。

`STM32/User/app_ui.c`
- 人机交互：
  - 按键扫描、消抖、短按长按事件
  - 参数设置逻辑（K1~K4）
  - OLED 三页显示 + 设置页显示
- 如果要改按键功能或页面文案，改这里。

`STM32/User/app_control.c`
- 执行器控制逻辑：
  - 风扇 AUTO/MAN 行为
  - 报警后风扇延时关断
  - 蜂鸣器鸣叫节拍

`STM32/User/app_ble.c`
- JDY23 蓝牙串口协议层：
  - 串口收包与按行解析
  - `GET/SET/SAVE/CLR` 命令处理
  - `$DATA/$CFG/$EVT/$LOG/$ACK` 回包
  - 报警日志记录与查询
- 如果要扩展手机命令，改这里最合适。

`STM32/User/stm32f10x_it.c/.h`
- 中断服务文件（当前业务主要仍在轮询主循环）。

`STM32/BSP/Delay.*`
- 微秒/毫秒延时封装。

`STM32/BSP/OLED.*`
- SSD1306 显示驱动（软件 I2C）。

### 24.3 修改建议（避免改乱）

- 改阈值范围：优先在 `app_ui.c`（按键）与 `app_ble.c`（蓝牙命令）保持一致。
- 改传感器采样策略：在 `app_sensor.c` 修改，尽量不碰 `main.c`。
- 改蓝牙协议字段：在 `app_ble.c` 修改，同时同步手机端 `index.vue` 解析逻辑。
- 改页面显示：在 `app_ui.c` 的 `Display_Task()` 改。
